Mes projets de développement

Pourquoi ?

Ma philosophie est que le meilleur moyen d'apprendre un langage ou une technique de développement est de concevoir puis réaliser un projet utile (au moins pour moi) et possédant un minimum de complexité ou de contraintes.

Étant curieux de nature et souhaitant parfaire mes compétences en JavaScript, je souhaitais connaître aussi bien ses facettes côté client que serveur. C'est pourquoi je me suis lancé dans le développement de l'application Open-dj.

Description du projet

Open dj est un serveur node.js qui à pour but de proposer une interface web style single page application, permettant à des utilisateurs mobiles/desktop de chercher et diffuser un flux audio en provenance de youtube, directement sur des hauts parleurs connectés sur la machine du serveur.

Une barre de bouton leur permet aussi de mettre en pause/lecture ou bien de contrôler le volume des hauts parleurs depuis le site.

Un système de playlist octroie à tous les utilisateurs, la possibilité de mettre en fil d'attente leur musique préférée s'il y en à déjà une en cours de diffusion.

La finalité de ce projet étant de pouvoir proposer à mes amis, lorsque je les reçois chez moi, de choisir eux même, avec leur smartphone, la musique qu'ils veulent écouter.

Je l'utilise surtout personnellement pour écouter de la musique pendant mes sessions de lecture.

Réalisation

Côté serveur, c'est node.js et express derrière un reverse proxy Nginx qui distribueront l'application aux clients. Un module maison s'occupera d'ouvrir des flux audio sur la machine hôte par l’intermédiaire du lecteur multimédia VLC.

Côté client, l'application est structurée sous forme d'un module principal AngularJS enveloppant la totalité du workflow de l'utilisateur, lui-même étant découpé en sous modules qui vont être responsables des fonctionnalités sur le site.

Une fois les clients sur Open-dj, une connexion websocket est établie entre leur navigateur et mon serveur. Toutes les transactions réalisées sur l'application passeront par leur connexion permanente au serveur Socket.IO ou par des promesses JavaScript avec le service $http de Angular.

Un champ de recherche offre la possibilité d'effectuer une recherche sur youtube.com, en fait, je récupère leur pattern de recherche sur le serveur et ouvre une requête http sur youtube via le module HTTP de node.js.

Une fois la requête terminée, je leur renvoie les résultats de recherche sur leur vue HTML sous forme de blocs cliquables.

Lorsqu'un utilisateur souhaite lancer la lecture d'une musique en cliquant sur un bloc de résultat, l'ID de la vidéo youtube est communiquée au serveur node.js. Tous les autres utilisateurs connectés au serveur websocket recevront une notification indiquant qu'un de leur homologue vient de lancer la lecture d'une musique.

Pendant ce temps-là, le serveur va construire une URL youtube contenant cet identifiant vidéo et le transmetra au stream de l'entrée standard d'un processus VLC en attente qui offre une API. Le processus du lecteur multimédia décryptera l'URL, récupérera et diffusera sur les hauts parleurs, le flux audio résultant de cette vidéo.

Le code source est libre et consultable sur mon repository github.

jean-massat.com V2

La deuxième version de jean-massat.com est celle sur laquelle vous vous trouvez actuellement.

L'hebergement provient d'une location de serveur VPS chez OVH, j'ai donc mis en place moi même un firewall avec iptables ainsi qu'un service fail2ban pour bannir les connexions malicieuses. J'ai utilisé le client LetsEncrypt pour vous proposer une connexion sécurisée via un certificat SSL généré par mes propres moyens, il est renouvellé tous les 30 jours grâce à un cronjob dans le crontab.

Pour l'architecture réseau, c'est un serveur Nginx qui s'occupe de traiter vos requêtes HTTP et les redirige, grâce à un reverse proxy, sur le processus node de l'application web qui écoute sur un port spécifique en interne. Enfin, j'ai utilisé le module npm PM2 pour persister l'application sur le serveur, de sorte à ce que, même après une eventuelle coupure des services OVH, le serveur web se remette automatiquement en place.

Le back-end est en node.js avec le framework web express.js, le module NPM «nodemailer» m'a permis de gérer l'envoi de vos messages sur ma boite mail à travers un formulaire web que vous pouvez retrouver en pied de page.

C'est une incrustation d'un template provenant de themeforest qui m'a permis de vous proposer ce design material et mobile responsive. J'ai du changer et supprimer un bon nombre de fonctionnalités pour le rendre le plus rapide possible pour le client.

Le code source du site est libre et consultable sur mon repository github.
